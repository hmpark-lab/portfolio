2. 프로젝트 개요

수행 기간: 2026. 04. 11 ~ 2026. 05.18 (약 5주)

분석 대상: 주요 카드 3사 (신한카드, KB국민카드, 우리카드) 주력 상품군

핵심 과제: 비정형 혜택 정보의 수치 정형화 및 전월 실적 연동형 혜택 계산 알고리즘 구현

개발 목적: 금융 소비자 정보 비대칭성 해소 및 데이터 기반의 합리적 소비 의사결정 지원

3. 배경 및 기존 서비스의 한계

현황: 기존 핀테크 앱의 카드 추천 서비스는 사용자의 주 지출 카테고리를 파악하여 해당 카테고리 혜택이 높은 카드를 우선적으로 추천함.

페르소나: 30대 직장인 김씨는 주변에서 혜택이 좋다고 입소문 난 신용카드를 최근 발급받았다. 각종 할인과 적립 혜택이 풍부하다는 설명에 기대가 컸지만, 사용한 지 몇 달이 지나자 생각보다 체감되는 이득은 크지 않았다. 월별 카드 명세서를 꼼꼼히 살펴본 결과, 실제 할인받은 금액은 이전에 사용하던 카드보다 오히려 적었다. 생활비 지출 구조와 맞지 않는 카드였던 탓이다.

한계점:

  • 복합 지출 구조 미반영: 특정 카테고리 외의 전체적인 지출 규모를 고려하지 않아 실제 피킹률 파악이 어려움.
  • 실적 조건의 간과: 카드사별 상이한 전월 실적 달성 여부를 시뮬레이션하지 않아 조건 미달 시 혜택이 0원이 되는 리스크를 노출함.
  • 할인 한도의 부재: 카테고리별 할인율만 강조하고 '통합 할인 한도'를 고려하지 않아, 실질적으로 체감하는 혜택 총액 산출이 불가능함.

본 프로젝트는 이러한 한계를 극복하기 위해 사용자의 '세부 지출액'과 카드사별 '실제 약관'을 정합시킨 정량적 혜택 계산 모델을 구축함.

5. 데이터 분석 및 수치 정형화 전략

비정형 데이터의 수치화

"스타벅스 20% 할인(월 최대 1만원)" → { brand: 'Starbucks', rate: 0.2, limit: 10000 } 와 같이 속성별로 분리하여 계산 가능한 형태로 가공합니다.

피킹률(Picking Rate) 분석

사용자의 총 지출액 대비 실제 할인/적립되는 금액의 비율을 계산하여, 단순 혜택액이 아닌 '소비 효율성'을 객관적 지표로 제시합니다.

6. 기대 효과 및 활용 방안

  • 데이터 분석 역량 증명: 비즈니스 로직을 알고리즘으로 모델링하고 데이터를 기반으로 분석하는 역량 입증
  • 정보 불균형 해소: 사용자가 인지하지 못한 카드사별 세부 제외 조항까지 반영한 정밀한 정보 제공
  • 플랫폼 확장성: 추후 카드사 API 연동을 통해 실시간 업데이트 서비스로 고도화 가능
